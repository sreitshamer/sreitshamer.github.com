arq_restore

An open-source command-line utility for restoring from backups created by Arq

Download a binary version here.

Get the source code here.

The data format is documented here.

Usage

First set 3 environment variables:

  • ARQ_ACCESS_KEY: your S3 access key ID
  • ARQ_SECRET_KEY: your S3 secret key
  • ARQ_ENCRYPTION_PASSWORD: the password you used to encrypt your backups

The s3 path value is of the form /<s3 bucket name>/<computer uuid>/buckets/<folder uuid>.

(The word "buckets" in the path should be "folders", but it's not for historical reasons).

Restore from a Backup

To restore the latest version of one of the folders, type the restore command listed.

For example, to restore the "src" folder from "Stefan's MacBook Pro" above, type:

arq_restore /akiaiyuk3n3tme6l4hfa.com.haystacksoftware.arq/32D9D7A2-3B3E-4BE7-B85B-0605AF24F570/buckets/1D142EAB-3218-48BD-BF5B-4EAEF504783E

Print Reflog

Each time Arq stores a new "commit" (backup), it writes a "reflog entry" containing the commit's key. You can print this reflog (it prints in reverse-chronological order) and then restore a specific commit instead of restoring the latest commit. The command is the same as the restore command for a folder, but with the "reflog" argument at the end. For example:

arq_restore /akiaiyuk3n3tme6l4hfa.com.haystacksoftware.arq/32D9D7A2-3B3E-4BE7-B85B-0605AF24F570/buckets/1D142EAB-3218-48BD-BF5B-4EAEF504783E reflog
